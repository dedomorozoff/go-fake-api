# Руководство по внесению вклада

Спасибо за ваш интерес к проекту Space API! Мы приветствуем любой вклад.

## Как внести вклад

### Сообщение об ошибках

Если вы нашли ошибку, пожалуйста:

1. Проверьте, не была ли она уже сообщена в Issues
2. Создайте новый Issue с подробным описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Go и ОС

### Предложение новых функций

1. Создайте Issue с описанием предлагаемой функции
2. Дождитесь обсуждения и одобрения
3. Создайте Pull Request

### Pull Requests

1. Fork репозитория
2. Создайте новую ветку (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Убедитесь, что код соответствует стандартам:
   ```bash
   go fmt ./...
   go vet ./...
   go test ./...
   ```
5. Commit изменений (`git commit -m 'Add some amazing feature'`)
6. Push в ветку (`git push origin feature/amazing-feature`)
7. Создайте Pull Request

## Стандарты кода

### Форматирование

- Используйте `go fmt` для форматирования кода
- Следуйте [Effective Go](https://golang.org/doc/effective_go.html)
- Используйте осмысленные имена переменных и функций

### Комментарии

- Добавляйте комментарии к экспортируемым функциям и типам
- Используйте godoc формат

### Тестирование

- Пишите тесты для новой функциональности
- Убедитесь, что все тесты проходят перед созданием PR
- Стремитесь к покрытию кода тестами > 70%

## Процесс разработки

1. Установите зависимости:
   ```bash
   go mod download
   ```

2. Запустите в режиме разработки:
   ```bash
   make dev  # с hot reload (требует air)
   # или
   make run  # без hot reload
   ```

3. Запустите тесты:
   ```bash
   make test
   ```

4. Соберите проект:
   ```bash
   make build
   ```
